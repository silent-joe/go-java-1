* Name of the application: "Music Addiction" (предварительное название).
* Application description: "Music Addiction" – рекомендательная сеть для прослушивания, хранения и обсуждения музыки.


* Actors: 
   - Гость.
   - Зарегистрированный пользователь.
   - Администратор.

* User story 1: Регистрация пользователя в системе.
Actor: Гость.
Use cases:
1) пользователь должен ввести имя.
2) пользователь должен ввести пароль.
3) после регистрации пользователь должен получить e-mail для активации своей учетной записи.
4) пользователь должен активировать учетную запись.

* User story 2: Вход в систему под ранее созданной учетной записью.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь должен ввести логин и пароль.

* User story 3: Поиск музыкальных записей в системе.
Actors: Гость, зарегистрированный пользователь.
Use cases:
1)	пользователь может искать музыкальные записи по ключевым словам.

* User story 4: Выбор музыкальной композиции по категориям.
Actors: Гость, зарегистрированный пользователь.
Use cases:
1)	пользователь может просматривать каталог записей по категориям (исполнитель/год выпуска/жанр/альбом/….)
2)	пользователь может искать музыкальные записи по ключевым словам в выбранной категории.

* User story 6: Составление пользовательских плейлистов.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь может составлять и сохранять свои плейлисты.
* User story 7: Прослушивание музыкальных записей в системе.
Actors: Гость, зарегистрированный пользователь.
Use cases:
1) пользователь может прослушать любую музыкальную запись, загруженную в систему.

* User story 8: Загрузка пользовательских музыкальных записей в систему.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь может создать альбом или папку для хранения своих музыкальных записей.
2) пользователь может выбрать куда загрузить музыкальную запись.
3) пользователь может загрузить музыкальную запись с локального диска в систему.

* User story 9: Написание комментариев к музыкальным записям.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь может оставить текстовый комментарий, длиной до 250 символов к музыкальной записи в системе.
2) пользователь может удалить свой комментарий. 

* User story 10: Оценивание музыкальных записей в системе.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь может поставить оценку от 1 до 10 любой музыкальной записи в системе.

 * User story 11: Управление данными пользователей системы.
Actor: Администратор.
Use cases:
1) администратор может удалить любую музыкальную запись/альбом/комментарий.
2) администратор может удалить учетную запись пользователя.

 * User story 12: Восстановление забытого пароля. 
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь должен ввести логин/e-mail, после чего получить пароль по e-mail.

* User story 13: Удаление учетной записи пользователя системы.
Actor: Зарегистрированный пользователь.
Use cases:
1) Пользователь может удалить свою учетную запись.

* User story 14: Изменение данных учетной записи.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь может изменить пароль.

 * User story 15: Настройка видимости личных данным профиля пользователя.
Actor: Зарегистрированный пользователь.
Use cases:
1) пользователь имеет возможность выбрать отображать/скрывать подробную информацию о себе.
