1. Состояние игры (класс GameSession). 
Части: 
Actor (персонажи, наемники, все кто представляет в игре человеческую расу, те, кто может инициировать действие).
Gear Все что можно отнести к классу снаряжение, обзательно принадлежит кому-то из Actor-ов, 
Software программное обеспечение, можно установить на Gear, если тот удовлетворяет требованиям. Подсистема (часть) Gear.
System (большой подраздел Gear) компьютерная система, можно купить за оченьдорого, можно создать из Gear под свои нужды (еще дороже), можно взломать и получить контроль.

2. Действие. (Action)
Cостояние системы изменяется в только случае, в результате команды Игрока. 
В нашем случае игроков будет два - Human (когда станет ммо - их будет больше, но все-равно Human) и AI. AI контроллирует все что нуждается в контроле и недоступно контролю Human. 
Взаимодействие будет вестись с помощью интерфейса Action. Игрок (Human или AI) инициирует Action, передает туда параметры - Кто действует и [с/над] Чем действует. 
Action меняет состояние игры (GameSession).
На любой Action есть свой навык, который влияет на результат действия (вероятностным образом, чем больше навык - тем больше вероятность успешного действия, или меньше вероятность что-то испортить).
Software еще не внедрил, еще не решил куда именно поставить. Отдельным интерфейсом, или как часть Gear (пока склоняюсь к Gear, это все-таки оборудование, хоть и нематериальное)

3. Ресурсы (интерфейс GameResources)
Одна часть ресурсов (те что можно купить (Purchaseable), нерасходуются при применении (Stable)) - это Gear (комплектующие для компьютеров, сами компьютеры, устройства для взлома/защиты/чего-то-еще). Основа, на которую наращивается "игровой функционал".
Другая часть ресурсов (Createable) - Software (Purchaseable, Stable, ),  который можно разрабатывать (develop). Основной вид ресурсов для действий. Нужна основа в виде Gear, которая удовлетворяет требованиям.
Треть часть - Datafiles (Createable, Mineable), которые можно будет создавать с помощью определенного Software (на любом Computable Gear). Нужны будут для ускорения, или вообще в принципе для улучшения софта.
Фактически вся сессия - это список Actor, Gear, System.

Еще мысли:
Software, который можно использовать - обзательно должен быть развернут (install) и стать частью System.
Software имеет уязвимость и умеет использовать уязвимость. Например, если атаковать защитную систему программой, что имеет уязвимость - защитная система может уничтожить твое ПО. (Ойвей, нелогично нудаладно, продумаю отдельно. Игра все-таки)
Вне системы - софт может быть, но как дистрибутив. (Например - DVD с софтом (Gear). Можно юзать installAction(Actor actor, Gear source, Gear target).

Кол-во навыков у персонажа не лимитировано. 
От неиспользования навыки деградируют до 1. 
Убрать навык совсем нельзя

Процесс взлома: 
0. Инициируем поиск систем в глобальном каталоге игры. (Местный интернет). Ка результат - изменится список список KnownSystems в Actor (или хранить это в рамках сессии, или хранить это в своем компьютере (Gear). Думаю).
1. Сканирование любой найденной системы. ScanAction.act(Actor actor, Gear tool, System target).
2. Как результат получим список Software, установленного на target.
3. Проанализируем (своей собственной головой) уязвимости target и собственные, чтобы защитное ПО системы не могло причинить нам вред, или минимизировать вред.
3. Выберем Software из списка ControlGear, которое умеет использовать уязвимости target, и само не уязвимо (минимально уязвимо) для защитных систем target и инициируем HackAction(Actor actor, Gear tool, System target).
4. Как результат получим boolean isSuccseed, и строку с результатми. Также изменится (веротяно) состояние ControlList. 
Аналогичным образом действуют другие игровые моменты. (цепочка из Action).

Возможные результаты действия Action.
1. Изменение PossesionsList - у тебя что-то прибудет/убудет (Datafiles, Money, Gear, Software). Отсюда никто кроме владельца ничего никогда не сможет убрать/добавить.
2. Изменение ControlList - ты получаешь доступ к чему-то, чем не владеешь (взломал/купил систему, нанял наемника). Включает в себя все из PossesionList. Но некоторыми штуками владеть нельзя, можно только контроллировать. И отсюда другие Игроки могут что-то удалить или добавить.
